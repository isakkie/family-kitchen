# 家庭厨房

一个家庭菜谱知识库原型。框架本身可以公开使用；菜谱库、冰箱、采购清单等个人数据默认只保存在使用者自己的浏览器本地。

## 已包含

- 菜谱库：查看、搜索、按标签筛选。
- 导入：粘贴菜谱正文后识别标题、用料、步骤、小贴士。
- 批量导入：支持 JSON、多个正文分隔导入，或一行一个链接先占位。
- 备份导出：将当前本地数据导出为 JSON。
- PWA：部署到网址后可在 iPhone / 安卓添加到桌面。
- 编辑：导入后可快速修改用料、步骤、标签和家庭备注。
- 冰箱：记录现有食材。
- 保鲜天数：新增食材时默认 3 天，也可以直接填 `1`、`5`、`10天`，系统会自动换算成到期日。
- 推荐你做：根据现有食材计算匹配度，显示缺少的主料。
- 采购：从菜谱或缺失食材生成购物清单。
- 菜单：安排一周菜单。

## 打开方式

当前本地预览地址：

```text
http://127.0.0.1:4174/index.html
```

也可以直接打开 `index.html` 使用。

## 数据与隐私

- 本地存储：菜谱库、冰箱、采购清单、一周菜单默认存在浏览器 `localStorage`，不会随代码仓库公开。
- 框架开放：这个仓库只保存应用框架和导入工具，不上传个人菜谱 JSON。
- 数据迁移：可以用“导出备份 / JSON 文件导入”在不同设备之间手动同步。
- 云端规划：后续可以增加可选云端接口。默认实现仍保持本地存储；有需要的人可以接自己的后端、第三方数据库或私有仓库。

当前版本没有内置账号系统，也没有把数据上传到任何服务器。

## 后续适合扩展

- 增加可选云端同步适配器，只给指定家庭或自建后端使用。
- 加入下厨房、小红书、公众号等来源的专用解析器。
- 支持 OCR 识别截图菜谱。
- 库存扣减和快过期提醒。
- 更细的食材别名库，例如“西红柿”和“番茄”合并。
